# FixThis v1.4.1

FixThis v1.4.1 is a release-readiness patch. It does not change the runtime
bridge, MCP queue schema, Gradle plugin behavior, or console workflow. It makes
the evidence path for source-trust and required-check promotion easier to audit
before maintainers tag or promote CI checks.

## Highlights

- **Source matching fixture lab discoverability.** The maintained fixture lab is
  now linked from the docs index and release-readiness dashboard, so maintainers
  can refresh shared-component, visual-area, interop-risk, recommended-edit, and
  confidence-cap evidence from one documented entry point.
- **Required-check readiness separation.** The readiness tracker now separates
  workflow observation, ready-to-require judgment, and actual enforcement. This
  prevents aggregate `ci.yml` success streaks from being mistaken for confirmed
  command-level GitHub status names.
- **Release-readiness guard coverage.** `check-release-readiness.mjs` now checks
  that the source matching fixture lab remains linked from the release evidence
  dashboard.
- **Planning trail.** The v1 trust evidence / required-checks umbrella design
  and implementation plan are recorded under historical planning docs for future
  follow-through.

## Compatibility

- **Runtime behavior:** unchanged.
- **Persisted MCP JSON:** unchanged. Compatibility fields such as `items`,
  `screens`, `itemId`, `screenId`, `targetEvidence`, `targetReliability`, and
  `sourceCandidates` keep their existing names and shapes.
- **Agent setup JSON:** unchanged from the v1.4 line.
- **Source-index schema:** unchanged.
- **Bridge protocol:** unchanged at `1.3`.
- **Gradle plugin:** `io.github.beyondwin.fixthis.compose` version `1.4.1`.
- **CLI/MCP package:** `fixthis-cli-mcp-v1.4.1.tar.gz`.
- **npm wrapper:** `@beyondwin/fixthis@1.4.1`.

## Agent install

From a Jetpack Compose Android app repository:

```bash
# macOS
brew install beyondwin/tools/fixthis

# npm
npm install -g @beyondwin/fixthis@1.4.1

# macOS/Linux fallback without package managers
curl -fsSL https://raw.githubusercontent.com/beyondwin/FixThis/main/scripts/install-fixthis.sh \
  | bash -s -- --version v1.4.1

fixthis install-agent --project-dir . --target all --verify --json
fixthis doctor --project-dir . --json
```

Manual Gradle plugin form:

```kotlin
plugins {
    id("io.github.beyondwin.fixthis.compose") version "1.4.1"
}
```

## Release evidence

This patch is documentation and release-operations scoped. Before publishing,
the release commit should pass the local release gate from
[`CONTRIBUTING.md`](../../CONTRIBUTING.md), especially:

```bash
npm run release:check
```

The release-readiness changes can also be checked directly with:

```bash
node scripts/check-release-readiness.mjs
npm run checks:observation:test
```
