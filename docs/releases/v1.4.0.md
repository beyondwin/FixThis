# FixThis v1.4.0

FixThis v1.4.0 turns the external-app handoff path into a more agent-readable
release surface. The release adds setup verification actions, connected Android
proof orchestration, runtime evidence attachments, Codex workflow packaging, and
role-specific edit-surface guidance so an agent can see what to do next instead
of guessing from a raw screenshot and source-candidate list.

## Highlights

- **Agent setup verification.** `fixthis install-agent --verify --json` now
  emits an agent setup report with `readiness`, `nextAction`,
  `readyForMcpTooling`, and an ordered `actions[]` queue. Agents can tell when
  they may open the feedback console, when the user must restart MCP tooling,
  and which command should be run next.
- **First-handoff autopilot evidence.** The agent-loop smoke path now checks
  that setup verification semantics lead to one MCP-readable sent feedback
  item, with recovery-oriented failure details when runtime prerequisites are
  missing.
- **Connected Android proof runner.** `npm run android:proof -- --strict`
  classifies SDK, ADB, device boot, app launch, Copy Prompt, agent-loop, and
  external fixture readiness in one local report, and `npm run release:gate`
  consumes that proof as a first-class release claim.
- **Runtime evidence attachments.** Feedback items can carry local runtime
  evidence summaries such as logcat windows, frame summaries, memory summaries,
  and trace artifact references without making external API calls or changing
  the persisted MCP queue contract.
- **Packaged Codex workflows.** The `.codex-plugin` package now includes
  install-agent, feedback-loop, Android evidence, and release-smoke workflow
  skills that wrap the existing FixThis commands instead of inventing a
  parallel path.
- **Role-specific edit-surface guidance.** Compact handoffs now include
  role/action guidance for likely edit surfaces, helping agents distinguish
  call-site edits, copy/data edits, component definitions, layout/style work,
  and interop boundary checks.

## Compatibility

- **V1 scope is unchanged.** FixThis remains debug-only, local/ADB-only, and
  Compose-only. This release does not add PyPI, Docker, or a new package
  channel.
- **Persisted MCP JSON:** additive only. The existing compatibility fields
  `items`, `screens`, `itemId`, `screenId`, `targetEvidence`,
  `targetReliability`, and `sourceCandidates` are unchanged.
- **Agent setup JSON:** setup verification reports now use schema `1.1` for the
  additive readiness/action contract.
- **Source-index schema:** current generated assets still use schema `1.3`.
- **Bridge protocol:** unchanged at `1.3`.
- **Gradle plugin:** `io.github.beyondwin.fixthis.compose` version `1.4.0`.
- **CLI/MCP package:** `fixthis-cli-mcp-v1.4.0.tar.gz`.
- **npm wrapper:** `@beyondwin/fixthis@1.4.0`.

## Agent install

From a Jetpack Compose Android app repository:

```bash
# macOS
brew install beyondwin/tools/fixthis

# npm
npm install -g @beyondwin/fixthis@1.4.0

# macOS/Linux fallback without package managers
curl -fsSL https://raw.githubusercontent.com/beyondwin/FixThis/main/scripts/install-fixthis.sh \
  | bash -s -- --version v1.4.0

fixthis install-agent --project-dir . --target all --verify --json
fixthis doctor --project-dir . --json
```

Manual Gradle plugin form:

```kotlin
plugins {
    id("io.github.beyondwin.fixthis.compose") version "1.4.0"
}
```

## Release evidence

The release commit should include passing or explicitly deferred evidence for
the first-handoff autopilot, connected Android proof, Android agent evidence
umbrella, external fixture matrix, release drift, and release gate sections in
[docs/contributing/release-readiness.md](../contributing/release-readiness.md).
Connected Android evidence remains local-only; if no unlocked emulator is
available, record the runtime commands as deferred rather than claiming
device-level proof.
