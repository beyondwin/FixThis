# FixThis v0.8.0

FixThis v0.8.0 sharpens Compose source matching for reusable components and
layout renderers, makes the feedback console push-first over SSE, and rounds
out agent setup with a Cursor MCP target and a machine-readable doctor
readiness contract.

## Highlights

- **Layout-renderer edit hints.** Source-index generation records a typed
  `LAYOUT_RENDERER` signal for `Layout(...)` and `SubcomposeLayout(...)` call
  sites inside composable owners. The matcher surfaces these as
  medium-confidence edit-surface hints when a selected target uses a strict
  `comp:<ComposableName>:...` test tag. The signal is conservative call-site
  evidence — it does not by itself make a source candidate high confidence.
- **Shared-component call-site ranking.** When a selected target's best
  candidate is a reusable composable definition invoked at two or more call
  sites, the handoff carries a `SHARED_COMPONENT` risk flag and a best-effort
  call-site inventory (`sourceCandidates[].callSites`). The inventory is ranked
  by a word-aware match between each call site's static context and the selected
  node's evidence, so the most plausible usage is listed first and flagged
  `mostLikely` (the feedback console labels it "(most likely)"). The console
  renders the inventory as a "Shared component used at" Evidence row.
- **Fallback-only console polling.** The feedback console is now SSE-first: a
  live SSE connection alone refreshes the screen, and visibility-resume / boot
  preview polling plus session polling only run when the event stream is
  disconnected. A healthy EventSource session issues zero `/api/sessions` and
  redundant `/preview` fetches, asserted by the browser reliability harness;
  explicit reconnect and manual refresh still resume polling on the fallback
  path.
- **Doctor `--json` readiness.** `fixthis doctor --json` now emits top-level
  `readiness` and `nextAction` fields, matching the first-run readiness contract
  shared by generated agent setup files and the feedback console. `install-agent
  --json` prefers `readiness.nextAction` for its top-level `nextAction`, so
  agents get one consistent next step after setup writes MCP config.
- **Cursor MCP target.** Agent setup now supports a Cursor MCP target:
  `--target cursor` writes a project-local `.cursor/mcp.json`, and `--target
  all` includes Cursor alongside Codex and Claude. The writer merges
  idempotently, preserving unrelated MCP servers and replacing only the fixthis
  entry, matching the Claude writer's semantics.

## Compatibility

- **Confidence stays capped at medium for shared components.** When the best
  source candidate is a reusable composable definition, FixThis caps it at
  medium confidence and cautions that editing the definition changes every
  usage, so the agent must verify the specific call site. Call sites are
  verification context only and do not raise the medium cap. Owner-function
  source evidence is likewise capped at medium with the `UNTYPED_FALLBACK` risk
  preserved when only untyped signals match.
- **Install paths are unchanged.** Homebrew, npm, the CLI/MCP GitHub Release
  package, the Gradle plugin id, and Maven Central coordinates are the same as
  the prior release; this cycle adds no new package channel.
- **Gradle plugin:** `io.github.beyondwin.fixthis.compose` version `0.7.0`.
- **CLI/MCP package:** `fixthis-cli-mcp-v0.7.0.tar.gz`.
- **npm wrapper:** `@beyondwin/fixthis@0.7.0`.
- **Bridge protocol:** unchanged at `1.3`.
- **Source-index schema:** `1.2`, preserving legacy fields while adding typed
  `signals` (now including `LAYOUT_RENDERER`).
- **Persisted MCP JSON:** additive only. `sourceCandidates[].callSites` and
  `editSurfaceCandidates[].role` are optional and older sessions omit them.
- **Android baseline:** `compileSdk` 34, `targetSdk` 34, `minSdk` 23.

## Agent install

From a Jetpack Compose Android app repository:

```bash
# macOS
brew install beyondwin/tools/fixthis

# npm
npm install -g @beyondwin/fixthis

# macOS/Linux fallback without package managers
curl -fsSL https://raw.githubusercontent.com/beyondwin/FixThis/main/scripts/install-fixthis.sh \
  | bash -s -- --version v0.7.0

fixthis install-agent --project-dir . --target all
fixthis doctor --project-dir . --json
```

Manual Gradle plugin form:

```kotlin
plugins {
    id("io.github.beyondwin.fixthis.compose") version "0.7.0"
}
```

## Release evidence

The release-prep commit captures source-matching, handoff-evaluation,
console-reliability, and readiness evidence per the v0.8 release claim manifest
in [docs/contributing/release-readiness.md](../contributing/release-readiness.md).
Connected device smoke remains local-only; if no unlocked emulator is
available, the runtime fixture command is recorded as deferred rather than
claiming device-level evidence.
