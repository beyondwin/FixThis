# FixThis v1.0.0

FixThis v1.0.0 is the first `1.0.0` external release. It deepens Compose source
matching so a wider set of selections resolves to the correct editable owner,
adds modifier-chain content-description evidence, and makes per-role edit-surface
confidence derive from — and explain — the underlying source match. These are
additive evidence improvements; the V1 scope is otherwise unchanged.

## Highlights

- **Lazy-list item mapping.** A selection inside a lazy-list item lambda
  (`items { ... }` / `itemsIndexed { ... }`) now remaps to the item composable
  instead of the enclosing list builder, so the agent starts from the editable
  row owner. It reuses existing evidence weights and does not by itself make a
  source candidate high confidence.
- **Navigation-destination mapping.** A selection inside a navigation
  destination lambda (`composable("route") { ... }`) now remaps to the
  destination composable instead of the enclosing graph builder. This is
  additive resolution evidence and does not on its own raise confidence.
- **Broadened layout-wrapper / slot recognition.** Custom Scaffold-style slot
  wrappers are now recognized across `content`, `topBar`, and `bottomBar` slots,
  so a node selected inside a wrapper surfaces the wrapper as an edit surface.
  The recognition is conservative and does not by itself make a source candidate
  high confidence.
- **contentDescription-via-semantics evidence.** An element whose
  `contentDescription` is supplied through a `.semantics { }` modifier chain is
  now treated as content-description evidence, matching the existing direct
  `contentDescription = ...` handling. It reuses existing evidence weights and
  is additive only.
- **Per-role edit-surface confidence with explainable basis.** Edit-surface
  confidence is no longer a flat per-role value: each candidate derives its
  confidence from the matched source candidate's own confidence, bounded by a
  per-role ceiling, and carries a short `basis=` string explaining why it earned
  it (e.g. "call site matched: …", "shared component definition: …", "interop
  boundary: …"). A weakly-matched source no longer presents as a strong edit
  surface. The `confidenceBasis` field is additive and absent on pre-existing
  sessions; this remains conservative evidence that does not by itself make a
  source candidate high confidence.

## Compatibility

- **V1 scope is unchanged.** This release remains debug-only, local/ADB-only,
  and Compose-only. The accumulated changes are additive evidence and matching
  improvements; none changes the supported runtime surface.
- **No new package channel.** PyPI and Docker are **not** added. Homebrew, the
  npm wrapper, the CLI/MCP GitHub Release package, the Gradle plugin id, and
  Maven Central coordinates are unchanged from the prior release.
- **Persisted MCP JSON:** additive only. `editSurfaceCandidates[].confidenceBasis`
  is optional and absent on older sessions. No breaking persisted-schema change
  lands in this release.
- **Bridge protocol:** unchanged. No wire-visible bridge-protocol change lands
  in this release; new source-matching outputs are additive signals.
- **Gradle plugin:** `io.github.beyondwin.fixthis.compose` version `1.0.0`.
- **CLI/MCP package:** `fixthis-cli-mcp-v1.0.0.tar.gz`.
- **npm wrapper:** `@beyondwin/fixthis@1.0.0`.

## Agent install

From a Jetpack Compose Android app repository:

```bash
# macOS
brew install beyondwin/tools/fixthis

# npm
npm install -g @beyondwin/fixthis@1.0.0

# macOS/Linux fallback without package managers
curl -fsSL https://raw.githubusercontent.com/beyondwin/FixThis/main/scripts/install-fixthis.sh \
  | bash -s -- --version v1.0.0

fixthis install-agent --project-dir . --target all
fixthis doctor --project-dir . --json
```

Manual Gradle plugin form:

```kotlin
plugins {
    id("io.github.beyondwin.fixthis.compose") version "1.0.0"
}
```
